### 2. Explain Socket Programming in Node.js, its architecture, communication process, major socket operations, and real-world use-cases.


>* Socket programming is a method of communication between two systems over a network where both systems can continuously send and receive data through a persistent connection.
>* A socket acts as an endpoint of communication between a client and a server.
>* Unlike traditional HTTP communication where a connection is repeatedly opened and closed, sockets maintain an active connection, allowing real-time and bidirectional communication.

Sockets are identified using:

* IP address
* Port number
* Communication protocol (TCP/UDP)

In Node.js, socket programming is commonly implemented using the built-in `net` module which provides TCP-based communication between systems.

---

### Socket Architecture

Socket communication mainly consists of two components:

**1. The Server** waits for incoming client connections and manages communication between systems.

* Create socket
* Bind socket to port
* Listen for clients
* Accept connections
* Send and receive data


```javascript id="q1sock"
const net = require('net');

const server = net.createServer();

server.listen(3000, () => {
    console.log("Server listening on port 3000");
});
```
**2. The Client** initiates communication by connecting to the server using the server’s IP address and port number.

```javascript id="q2sock"
const net = require('net');

const client = net.createConnection({
    host: 'localhost',
    port: 3000
});
```
---

### Socket Connection Process

* The server detects new connections using the `"connection"` event.

Example:

```javascript id="q3sock"
server.on('connection', (socket) => {
    console.log("New client connected");
});
```
* The client sends data using the `write()` method.

Example:

```javascript id="q4sock"
client.write("Hello Server");
```

* The server receives incoming data using the `"data"` event.

Example:

```javascript id="q5sock"
socket.on("data", (chunk) => {
    console.log(chunk.toString());
});
```

* The server can also respond back to the client.

Example:

```javascript id="q6sock"
socket.write("Hello Client");
```

* Socket communication is stream-based, meaning messages may arrive in multiple chunks rather than one complete packet.


```javascript id="q7sock"
let fullData = "";

socket.on("data", (chunk) => {
    fullData += chunk.toString();
});
```

This allows data to look the same as the original.

---

### Resilient Servers

Servers must handle:

* Client disconnects
* Runtime errors
* Broken connections
* Unexpected failures

Example of error handling:

```javascript id="q8sock"
socket.on("error", (err) => {
    console.log("Error:", err.message);
});
```
---

### File Upload using Sockets

Sockets can also transfer files because files are streams of binary data.

### Client Side

The client reads the file using a readable stream and sends chunks through the socket.

Example:

```javascript id="q10sock"
const fs = require('fs');

const readStream =
    fs.createReadStream("input.txt");

readStream.on("data", (chunk) => {
    client.write(chunk);
});
```

---

### Server Side

The server receives chunks and writes them into a file.

Example:

```javascript id="q11sock"
const fs = require('fs');

const writeStream =
    fs.createWriteStream("output.txt");

socket.on("data", (chunk) => {
    writeStream.write(chunk);
});
```

### Advantages of Socket Programming

* Supports real-time communication
* Persistent connection between systems
* Faster than repeated HTTP requests
* Efficient handling of streaming data
* Enables bidirectional communication
* Efficient for continuous data transfer
* Useful for large-scale distributed systems

---
### Important Socket Events

| Event          | Purpose                              |
| -------------- | ------------------------------------ |
| `"connection"` | Triggered when client connects       |
| `"data"`       | Triggered when data arrives          |
| `"end"`        | Triggered when transmission finishes |
| `"close"`      | Triggered when socket closes         |
| `"error"`      | Triggered when error occurs          |

### Important Socket Methods

| Method               | Purpose            |
| -------------------- | ------------------ |
| `createServer()`     | Creates TCP server |
| `listen()`           | Starts server      |
| `createConnection()` | Connects client    |
| `write()`            | Sends data         |
| `end()`              | Closes connection  |
